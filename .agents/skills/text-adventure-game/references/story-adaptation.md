# 改编工作流:用户自带大纲 / 完整故事 → 互动游戏

> 入口:用户**不是给一句话题材**,而是带来了自己写的大纲、章节梗概、或完整成文故事。
> 本文是阶段 1-2 的**改编替代路径**(其余阶段照常):canon 从原文**提取**而非自创、剧情**忠于原作**、
> 互动化是「给既有故事加选择与后果」,不是借题重写。
> 方法论依据(调研双源核实):Sam Kabo Ashwell《Standard Patterns in Choice-Based Games》(分支结构正典)、
> Choice of Games《By the Numbers》(delayed branching)、Ryan North《To Be or Not To Be》/ Dave Morris
> 《Frankenstein》/ inkle《80 Days》(改编忠实度三档先例)。

## 0. 素材怎么进来

- **短大纲(几百字)**:直接贴在对话里即可。
- **长文(章节梗概/完整故事)**:让用户放到**项目根 `story.md`**(或任意 .md/.txt),然后 `/new-game 改编 story.md`。
  你先**通读全文**(长文分段读完,别只读开头),再开工;canon 提取时引用原文行号/章节号留痕。
- 用户只有零散设定+人物表也算改编素材:同样走本文,缺的剧情再与用户共创。

## 1. 先问一个问题:忠实度选档(别自作主张)

改编有三档成熟先例,**风格差异巨大,必须让用户选**(默认 A):

| 档 | 做法 | 先例 |
|---|---|---|
| **A. 正典线 + 增殖分支**(默认) | 原作剧情完整保留为一条**保证可达的主线**;新分支/新结局做增量,玩家可以全程走"原著路线" | Ryan North《To Be or Not To Be》:莎翁原剧情每步用骷髅图标标出,另加 110 个新结局 |
| **B. 重写浓缩 + 保骨架** | 保留情节骨架与名场面,但全文**重写成互动节奏**(每屏 100-200 词);适合原文是小说体、直接搬会满屏 wall of text | Dave Morris《Frankenstein》:浓缩 Shelley 散文但保留风味,高光场景(冰川相会)保原句味道 |
| **C. 松改编** | 只保留前提/目标/世界观,情节让位给系统与探索;**这是再创作**,必须用户明说才走 | inkle《80 Days》:"与原作的对话",城市 7→150+,有意翻转原作的阶级叙事 |

**结局守则**(三档先例的共同点):原作结局**永远保留且可达**(A 档=正典结局、B 档=主结局);
新增分支结局允许且常是卖点,但别动原作结局本身。**别为分支而分支**——零选择的 kinetic 段落也是
合法形态,原文张力强的章节可以整段线性推进(中间只放表态型选择)。

## 2. 结构选型:故事形态 → 分支模式(Ashwell 八模式速查)

| 原作形态 | 选这个结构 | 在 Amatlas 里长什么样 |
|---|---|---|
| 强情节、单主角(多数小说) | **Gauntlet / Branch-and-Bottleneck** | 正典主线=每章 map 的无条件主干出口;选择写 flag,后章 look/出口按 flag 回写与分化 |
| 旅程 / 公路 / episodic | **Quest**(按**地理**组织) | 章=map、地点=节点簇,簇内多解法;主干=跨图出口 |
| 调查 / 解谜 / 走访章节 | **局部 storylet**(乱序信息) | hub 节点 + 多条 links 任意顺序探;线索=flag,集齐开门(`requires`) |
| 群像 / 多视角 | **Sorting Hat**(线路制) | 前期共享章 → 按累积 flag 分流进各自 route(=各自的 map 链);分流要让玩家**感知到** |
| 时间循环 / 日常推进 | **Loop and Grow** | 主线绕回 hub,flag 解锁新 links(给"重复"一个叙事理由) |
| ⚠️ 任何长篇 | **别用 Time Cave**(纯发散树) | 组合爆炸:每个分支都要写全文,主干被平行宇宙稀释——改编大忌 |

## 3. 选择点挖掘:线性场景的哪些时刻天然是选择(五类)

1. **决策/两难**:原文里主角犹豫、权衡、犯错的时刻=现成选择点。每个选项都要值得选,
   "坏决定通向更有趣的故事"(80 Days 主笔:作者的工作是引诱玩家做坏决定)。
2. **对话**:先定场景**潜台词**再造选项;accept / reject / deflect 三态;说错话可以直接掉出对话
   (trapdoor)。对话分支结构本身就是人物塑造(inkle《Sparkling Dialogue》)。
3. **信息顺序**:原文的回忆/调查/档案段 → 拆成可乱序触达的节点(见 §2 storylet 行),玩家自选探索顺序。
4. **空间**:原文的赶路/探访 → 地图化,路线选择本身成为玩法。
5. **表态(reflective)**:不改剧情、只表达主角内心的选择——**合法且重要**(玩家借此贴近既定主角);
   无需后果,但**至少回应一行**(look 一句变化 / 一条 flag 供后文 callback)。

**核心工艺 = delayed branching**(Choice of Games 正统):早期选择**不立即分叉剧情**,而是写
flag/数值,决定**后续章节中事件的结果**——章节序列保持原作线性,体验却高度互动。配套铁律:
**每章都要有回写**(用 look 函数按 flag 给一句"你当初…所以现在…"的 callback)——只切碎不回写=
"一堆不影响剧情的小按钮"(CoG 点名的头号改编失败模式);只合并不回写=互动性被掠夺。
假选择(同 to)合法条件:有回应(一行文本/一个 flag)或纯表态;零承认的同归=欺骗。

## 4. 改编版 canon 规则(阶段 1 的差异)

- canon **从原文提取**:时间线/人物/世界规则逐条带**原文出处**(章节/行号);没写的才标「补:×××(与用户确认)」。
- 多一节「**正典事件序**」:原作关键事件按序编号——这是 §2 主线的蓝图,也是阶段 4 验收"正典线可达"的清单。
- 人物声音速写直接摘原文台词风格;B 档重写时拿它对味。
- 原文照搬进 look 时注意互动节奏:**每屏 100-200 词,超 400 词必拆**(拆成"继续"链或并入选择)。

## 5. 工作流(替换阶段 1-2 的开头,其余照常)

1. 通读素材全文 → 提取式 canon(含正典事件序)。
2. **问用户两件事**:忠实度选档(§1)+ 是否允许新增分支结局。
3. 按 §2 选结构 → 地图骨架(沿用「先世界后填充」纪律:章→map、场景→节点、正典线=主干出口)。
4. 逐章改造:正典文本入 look(按档位决定照搬/浓缩)→ §3 挖选择点 → delayed branching 布 flag 与回写。
5. 阶段 3-5(装配/审计/构建)照常;阶段 4 **额外验收**:沿正典事件序走一遍主线,确认原作路线完整可达、
   原作结局可达(graph-audit 的可达性只保证图连通,"正典线完整"要按事件序人工/自动走查)。
