对 $ARGUMENTS 做"重访文字一致性"专项审查。这是分支叙事游戏最易出戏、脚本最难发现的一类问题。方法见 `references/revisit-consistency-audit.md`。

## 步骤
0. **读 `world.js` 取材**:数每个节点的入边(指向它的 `links[].to` / `exits[].to`),找高入度(多入口)节点 = 重访候选;再逐个读 `look` 正文核。
1. **算入度 + 可重入性**:解析图,找出"多入口(in-degree ≥ 2)且在环里可重入"的节点。
2. **扫一次性措辞**:这些节点的 `look` 正文是否写死了:
   - "你**第一次**看到 X" / "你**终于**到达" / "一个你**没见过**的人迎上来"
   - NPC **重新自我介绍** / 把玩家当**新人**对话(玩家可能早就认识了)
   - **一次性动作**:"你花了两小时**搭建**营地" / "你**初次**踏入"
   - **固定时点**:写死"傍晚/日落/第N天",但玩家可能在别的时间重访
3. **逐个读正文 + 看入口来路**,判断真问题 vs 伪问题:
   - 真问题:多入口、未分首次/重访、来路时间/认知状态不一致。
   - 伪问题:章节切换点(单向推进的"天亮了"合理)、回忆性"第一次"(不是到达节点)、已分流的。

## 修法(统一)
- 用 `look:{ first:'首次版(完整初到 + 介绍 + 一次性动作 + 时间描写)', return:'重访版(简短"你回到了X")' }`,或函数 `look:(S, first) => first ? 首次 : 重访`。
- **时间描写只放首次分支**(避免重访时时间倒流)。
- ⚠️ `look:{first,return}` 的首次/重访由**引擎访问计数**(`firstTime()`,首次=visits≤1)自动判定,不用手设 flag——正好避开旧坑。**若**你在 `look` 函数里手动按 flag 分首次,别用"节点一进入就设的 flag"(那首次进入就被设了,逻辑会错);用"经历过初次剧情才有"的 flag(如判断村庄首次用 `met_chief` 而非 `arrived_village`)。

## 报告
列出:节点名、入口数、写死的原句、对应修法。区分真问题/伪问题。
