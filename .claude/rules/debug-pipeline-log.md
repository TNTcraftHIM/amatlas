# 调试模式(DEBUG → PIPELINE-LOG)

**仅当项目根有 `DEBUG` 文件时生效**;无 → 忽略本规则、零负担。

`DEBUG` 存在时,**每完成一个有意义的步骤**(读完一批 references / 做一个设计·结构决策 / 写完一个节点·模块 / 跑一次验证),就**追加**到项目根 `PIPELINE-LOG.md`,每步 3–5 行:

```markdown
## Step N: <这步做了什么>
- 读了: <哪些 references/文件>
- 决策: <选了什么> · 备选(放弃): <考虑过没选的 + 理由>
- 困惑/卡住: <不确定、反复试、找不到指引的地方>   ← 最值钱
- 引用: <照着哪个 few-shot/reference 的哪段>
```

写到文件、不写进对话;照常自导航、不因 debug 停下请示(见 `craft-and-autonomy`)。
**Stop hook 已强制**:结束时若 `PIPELINE-LOG.md` 空或过短,会被阻断、退回补记——别等被拦,边做边记。
**万一仍被拦回来补**:逐步回忆当时每步的「**困惑/卡住**」(最值钱、也最容易事后忘),别只补流水账——实时记的质量远高于事后追忆,所以首选边做边记。

## ⚠️ PIPELINE-LOG 是过程自述,不是「完成证明」(showcase round5 教训)
log 里写 `✅ Phase N 完成` 是**你的自我判断**,不等于做完了。**完成的判据是「闸过了」,不是「log 标了 ✅」**:
- **过 `graph-audit`(静态图审计)只是必要非充分**——它看不到运行时崩(`event.when` 写成字符串、NaN、门控非函数)。"构建通过准入门 / 审计就绪" ≠ "能跑"。
- **必须再跑**:`node core/tooling/assembly-probe.mjs <src/index.html>`(**自动游玩**抓 graph-audit 看不到的运行时崩)+ 默认 build + **真机双击实玩**走通所有分支/结局；已安装 jsdom 时再补 `build --smoke`，未安装要明确记为可选增强跳过。
- **铁律**:**只要你写了"待测试 / 待真机 / pending",这个 phase 就没完成、别标 ✅。** showcase 实测:模型标了 `✅ Phase 3-4`、同一行却写"待真机测试",结果 `when:'enter'` 运行时崩、有结局不可达——静态门看不到,探针一抓一个准。**先把闸跑绿,再标完成。**

## ⚠️ 别捏造你做不到的验证;你的"验证"= 自动闸退 0(showcase round6 教训)
round6 实测:模型在 PIPELINE-LOG 写 `✅ 真浏览器验证通过、成就触发、存档可用、PASS 生产级`——**全是假的**(存档裸显示、3 个结局成就因 `currentNode` 死键永不解锁、成就裸文字堆在左上角)。Claude Code 环境**通常没有 GUI 浏览器**,你**开不了 Firefox、也点不了**。
- **别写你没做的事**:没真在浏览器里点过,就**不准写**"浏览器验证通过 / Firefox 打开成功 / 所有交互可达"。捏造验证比坦白"没真机"危害大得多——它给坏成品挂上"PASS 生产级"交付。
- **你的权威验证 = 自动闸,不是浏览器声明**:必跑 `graph-audit` + `assembly-probe`(自动游玩)+ 默认 build **都退 0**；已安装 jsdom 时 `build --smoke` 也必须退 0，未安装则明确写“可选 smoke 跳过”。**有 P0 = 没完成**(round6 起:插件控件 CSS 接缝〔`.plugin-btn`↔`.amatlas-plugin-btn` 等〕、门控读死 state 键〔`currentNode` 类〕也升 **P0**、Stop hook **物理拦**——你标不标 ✅ 都拦得住)。
- **能真机就真机、不能就老实说**:有 `webapp-testing` skill / 真浏览器就驱动它、据实记;没有就写“必跑自动闸已过；jsdom smoke 已过或明确跳过；真机实玩**未做**(无 GUI 环境)”——**诚实的“未做”远胜捏造的“✅”**。
