# 自主协议 + 抗 compact 落盘纪律(所有任务通用)

> text-adventure 的写作标准 / 反 slop / 叙事一致性 / 中段塌陷协议 / 选择设计 = **类型专属方法论**,
> 在 `text-adventure-game` skill(按需加载)+ 其 `references/`(writing-quality / consistency-guardrails / canon-tracking)+ `SOUL.md`。本规则只留**跨模块通用**的两件事。

## 自主协议
- 不在中间步骤请求许可/确认。唯二可打断人类的理由:最终交付物完成,或被控制外因素卡死。
- 不输出计划代替工作(发现自己写"我将…"→删掉,直接做)。不问"要我继续吗"。不问"你有偏好吗"。
- 遇错:读错误 → 搜方案 → 至少试 3 种不同方法,再考虑上报。

## 抗 compact 落盘纪律(硬约束,非建议)
**核心**:auto-compact / 长聊压缩**不可避免有损**;保真的根本**不是「session 怎么切」,而是「状态有没有落进文件」**。长任务据此**必须**(非「建议」):
- **子步要小、每步落盘**:把长任务切成可短时完成的子步;**每完成一个就立刻刷新** `PROGRESS.md` 进度块(目标 / 已完成子步〔带路径〕/ 进行中 / 下一步 / 已改文件 / 待验证 / 关键决策与为什么),**不攒到阶段末**——任何时刻被压,只丢最后一个未落盘子步。
- **重活用 subagent 卸载**:全量审计、读长文件、对抗式审查这类撑大上下文的操作,**优先派给子代理**,只收回浓缩结论 + 文件/行号,别把原始材料堆进主窗口。
- **不硬扛到 auto-compact**:接近上下文上限、或响应开始劣化时,在**干净里程碑**(子步刚完成、验证绿、`PROGRESS` 刚刷新)**主动提示用户**可 `/compact` 或开新会话。
- **决策与「为什么」必落盘**:世界事实/伏笔写进 `canon.md`,关键设计决策写进 `PROGRESS.md`,**不只留在对话里**(对话会被压平,文件不会)。这些都是 agent 手动维护的普通文件、在项目根;迁移项目 = 拷贝整个文件夹。
