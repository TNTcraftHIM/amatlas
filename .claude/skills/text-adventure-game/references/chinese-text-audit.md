# 中文叙事文本审计模式

从《制图师的挽歌》668KB 中文文本全文手工审读中提取的系统性错误模式。
适用于所有 LLM 生成的中文创意写作。

## 审计方法论

**不用脚本/正则偷懒。** 逐段手工审读，每 150-200 行一组。
正则只能捕获结构性错误（逗号截断、编码损坏），语义错误必须靠人脑判断。
用户明确要求："一行行检查，不允许使用工具（可以每检查完一段记一下笔记防止上下文遗忘）"。
这是核心工作流偏好——grep/sed 只用于验证修复结果，不用于发现语义问题。

**笔记防遗忘：** 每审完一段（200 行），在上下文中记一下：
- 发现了什么问题（行号 + 描述）
- 修了什么
- 还没修什么

**审读顺序：**
1. 节点文本（`world.js` 的 `maps.<map>.nodes.<node>.look`）— 从 `start` 节点开始逐段读
2. 多片段世界的合并文本 — 大型游戏拆多个 `world.js` 片段、构建前合并，注意片段间的重复/矛盾
3. 首次/重访分支文本（`look:{first,return}` 或 `look:(S,first)=>…` 函数）— 确认语义在两种访问态下都通顺

## 15 种错误模式

### A. 缺判断词「是」（最高频）

模式：`NP₁ 的 NP₂` → 应为 `NP₁ 的是 NP₂`

典型位置：让步句（「但让你停下脚步的X」）、判断句（「那一个X」）、
系表句（「他的表情X的Y」）

实例：
```
❌ 但让你停下脚步的它的形状
✅ 但让你停下脚步的是它的形状

❌ 那一个"形状"
✅ 那是一个"形状"

❌ 那些图案一幅地图
✅ 那些图案是一幅地图

❌ 它凝固的液体
✅ 它是凝固的液体

❌ 他的表情深沉的平静
✅ 他的表情是深沉的平静

❌ 波纹的方向从岸边向外海扩散的
✅ 波纹的方向是从岸边向外海扩散的
```

### B. 缺连接词「而是」

模式：`不是X，Y` → 应为 `不是X，而是Y`

中文「不是X，而是Y」是标准对比句式。LLM 频繁省略「而是」，
导致句子读起来像两个不相关的陈述并列。

实例：
```
❌ 不是苦涩，一种更复杂的情感
✅ 不是苦涩，是一种更复杂的情感

❌ 不是担忧，一种古老的、深沉的理解
✅ 不是担忧，而是一种古老的、深沉的理解

❌ 不是控制岛屿，也成为岛屿的一部分
✅ 不是控制岛屿，而是成为岛屿的一部分

❌ 不是蓝光，一种温暖的、金色的光
✅ 不是蓝光，而是一种温暖的、金色的光

❌ 不是图像，一种感觉
✅ 不是图像，而是一种感觉

❌ 不是审视，纯粹的理解
✅ 不是审视，而是纯粹的理解
```

**短句豁免：** `不是恐惧，是激动` — 两词对比中省略「而是」是可接受的口语。
判断标准：如果 Y 是一个名词短语（≥4 字），通常需要「而是」。

### C. 逗号截断

模式：句子以「，」结尾，内容不完整。

原因：LLM 在生成过程中被截断，或多片段合并/从旧格式迁移 `look` 文本时丢失尾部。

```
❌ 找不到干燥的木头，
✅ 找不到干燥的木头。

❌ 苔藓的湿气渗进衣服，
✅ 苔藓的湿气渗进衣服。

❌ 蹲在边缘，开始计时，
✅ 你蹲在边缘，开始计时。
```

**检测：** 在源数据里查 `grep '，"' world.js`（`look` 字符串以「，」结尾就紧跟收尾引号），
也可在构建产物里查 `grep '，</p>' dist.html`（present-dom 把每行包成 `<p class="line-*">`）。

**引号/标签陷阱：** 有些逗号截断紧贴字符串收尾引号或 `</p>` 标签（如 `渗进衣服，"` / `渗进衣服，</p>`），
按行尾匹配时会漏掉（因为行尾是引号或标签，不是逗号）。
修复直接改 `world.js` 的 `look` 文本（句号结尾），改完重新 `node pipeline/build/build.mjs src/index.html` 构建。

### D. 文字重复

同段内或同句内同一短语出现两次。

```
❌ 海风从南边吹来。海滩高处...海风从南边吹来，这个位置
✅ 海风从南边吹来。海滩高处...这个位置

❌ 它是用来被画的——它是活的——不是用来被画的，是用来被感受的
✅ 它是活的——不是用来被画的，是用来被感受的

❌ 发出了一声一种奇异的声音
✅ 发出了一种奇异的声音
```

### E. POV 漂移

角色在对话中用第三人称指代自己。

```
❌ 「每一个制图师都有这个选择。雷纳选择了融合。」（说话的人就是雷纳）
✅ 「每一个制图师都有这个选择。我选择了融合。」
```

### F. 感官混搭

隐喻中混合不同感官维度，无过渡。

```
❌ 不是音乐，比文明更久远的光（听觉→视觉）
✅ 不是音乐——比任何旋律都更古老（统一为听觉）
```

### G. 路由字段错误（链接副作用写错形式）

链接的副作用必须写在 `run:(S)=>{…}` 函数体里（直接改 `S.flags`/属性），
不是某种声明式的 `flags_set`/`effects` 对象——模块化呈现器不解析那些，写了不生效。

```
❌ { label: '跟上雷纳', to: '…', flags_set: { following_renar: true } }
✅ { label: '跟上雷纳', to: '…', run: function (S) { S.flags.following_renar = true; } }
```

合并多个 `world.js` 片段、或从旧格式迁移时易把声明式字段带回来。
**检测：** `grep 'flags_set\|effects:' world.js` 应返回 0 结果（副作用一律走 `run`）。

### H. 编码损坏

中文字符被截断为乱码。

```
❌ 鞋底粘着一种黑色的、�ite亮的物质
✅ 鞋底粘着一种黑色的、锃亮的物质
```

**检测：** `grep '�' world.js` 应返回 0 结果（源数据干净，构建产物自然干净）。

### I. 残缺句（缺谓语/缺主语）

```
❌ 他们我预想的那副模样
✅ 他们不是我预想的那副模样

❌ 它记录的运动。不是形状
✅ 它记录的是运动。不是形状

❌ 那些向外延伸的线流动的方向
✅ 那些向外延伸的线指示着流动的方向
```

### J. 谓语缺失（「也」前缺「是」）

```
❌ 不是快乐，也更深沉的东西
✅ 不是快乐，而是更深沉的东西

❌ 不是被吞噬，两条河流汇入
✅ 不是被吞噬，而是两条河流汇入
```

### K. 多片段合并引入旧 bug

大型游戏拆多个 `world.js` 片段、构建前合并。如果某个片段里残留已在别处修过的
bug（如把副作用写成 `flags_set` 而非 `run`），合并后会把旧 bug 带回来；
同一节点在不同片段重复定义 `look`/`links` 时，后定义的也可能盖掉已修好的文本。

**修复后必须全（合并后）文件搜索确认：**
```bash
grep 'flags_set' world.js   # 应为 0
grep '�' world.js           # 应为 0
```

**重要：每次修复后都要重新验证。** 不要假设一次修复覆盖了所有实例。
特别是 `flags_set` 和 `不是X，Y` 模式——多个片段中可能有多个实例，
用 `replace_all=true` 修了一批后，必须再次 grep 确认为 0。
本次审计中 `不是控制岛屿，也成为` 出现了 2 次，`不是蓝光，一种` 出现了 2 次，
都必须用 `replace_all=true` 一次性修完。

### L. 量词/助词重叠

```
❌ 发出了一声一种奇异的声音（「一声」+「一种」重叠）
✅ 发出了一种奇异的声音

❌ 但一件事是确定的（缺量词「有一」）
✅ 但有一件事是确定的
```

### M. 物品/概念命名过早

物品在叙事上还没有被正式命名时，文本就已经使用了正式名称。

**典型场景：** 墨水在 prologue 节点里叫"陌生的墨水"，但后续 `beach_awakening`、
`shipwreck_prepare`、`camp_setup` 等节点的 `look` 文本中直接叫"活墨水"——
而"活墨水"这个名字要到 `village_market` 节点卡伊说"我们的墨水是活的"才第一次出现。

**检测：**
1. 找到物品正式命名的节点（玩家第一次在 `look` 里读到正式名的地方）
2. 搜索该名称在所有节点 `look` 文本中的出现，按叙事时序排查更早的节点
3. 排除 `run` 函数体里的 `S.inventory.push(...)`（代码，不显示给玩家）
4. 只检查 `look` 字符串中的玩家可见文本

**修复：** 命名前用通用称呼：
- "那瓶墨水"（指代自己的物品）
- "那种墨水"（指代见过的东西）
- "雷纳的墨水"（强调来源）
- "那种东西"（泛指）

命名后用正式名称"活墨水"。

**注意：** `run: (S) => S.inventory.push('活墨水')` 在命名前的节点中是正确的——
这是物品 ID，不是玩家看到的文本。

### N. "一种"过度使用

LLM 生成中文时高频使用"一种"作为量词/修饰词。
10 万字中出现 400+ 次是典型频率，密集时显得单调。

**可安全替换的模式（约占 30%）：**
- `带着一种甜腻的` → `带着甜腻的`
- `一种微弱的、脉动的光` → `微弱的、脉动的光`
- `形成了一种图案` → `形成了图案`
- `产生一种共鸣` → `产生共鸣`
- `一种遗憾涌上来` → `遗憾涌上来`
- `以一种你无法理解的方式` → `以你无法理解的方式`
- `一种复杂的情绪` → `复杂的情绪`
- `感觉到一种敬畏` → `感觉到敬畏`

**不应替换的模式（"一种"是语义必要的）：**
- `一种新的东西`（强调"新类型"）
- `一种流动的记录`（定义新概念）
- `一种你和岛屿一起创造的东西`（强调独特性）

**目标：** 从 400+ 降到 300 以下（减 25%+）。
**方法：** 批量替换最机械的用法，保留语义必要的实例。
**验证：** `grep -c "一种" world.js` 确认数量。

### O. 条件文本标记泄漏给玩家

模块化呈现器**不解析** `{{}}` 标记——任何 `{{if:…}}`/`{{else}}`/`{{endif}}` 写进
`look` 文本都会**原样显示**给玩家。条件/首次重访分支不该用文本内标记表达，
而应交给引擎：`look:{first:'首次',return:'重访'}`（引擎按访问计数自动选），
或 `look:(S,first)=> first ? '…' : (S.flags.X ? '…' : '…')`（函数里用 JS 条件分支）。

**症状：** 玩家看到 `{{else}}你在岛上其他地方见过的{{endif}}`（从旧格式迁移时漏改）。

**修复：** 把条件文本从 `look` 字符串里拆出来，改写成 `look` 的 `{first,return}` 对象
或函数形式；全文件 `grep '{{' world.js` 应返回 0 结果。两种访问态都要逐一读通顺。

## 审计检查清单

按以下顺序逐项检查：

1. [ ] `grep '�' world.js` → 0 结果（编码损坏）
2. [ ] `grep 'flags_set\|effects:' world.js` → 0 结果（路由字段错误，副作用一律走 `run`）
3. [ ] `grep '，"' world.js` → 逐一检查（`look` 文本逗号截断）
4. [ ] `grep '{{' world.js` → 0 结果（条件文本标记泄漏）
5. [ ] `grep -c "一种" world.js` → 记录数量，目标 <300
6. [ ] 手工审读每个节点 `look` 的前 2 句（空间/人物定位）；`{first,return}` 两态都要读
7. [ ] 手工审读所有「不是X，Y」结构（缺「而是」）
8. [ ] 手工审读所有「NP₁ 的 NP₂」结构（缺「是」）
9. [ ] 检查多片段合并后是否有节点被重复定义、引入旧 bug
10. [ ] 检查所有 NPC 对话的 POV（第三人称自指）
11. [ ] 检查隐喻的感官维度一致性
